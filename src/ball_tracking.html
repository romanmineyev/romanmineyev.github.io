Perception-Based Football Tracking with a 3-DOF Gimbal Mounted on a Quadruped

Overview

Quadrupedal robots are becoming capable of completing increasingly complex tasks, ranging from warehouse maintenance tasks to parkour over uneven terrains. Object-catching with quadrupeds, however, is something that has been relatively less explored. This project explores the capabilities of a quadrupedal robot in this domain.

The long-term goal of the project is as follows. Given a human football player, we would like a quadrupedal robot to run a football route and catch the ball. The quadruped will catch this ball with its two robotic arms, attached to its base. The tracking is be done using a camera, also attached to the base of the Spot robot. The camera is mounted on top of a 3-DOF module, giving us the freedom to track the football from almost any angle.

Methodology

Fig 1: Hardware setup of football-catching robot

To catch the football, I used the Spot robot from Boston Dynamics. The robot is equipped with a Realsense D435i attached to a 3-DOF gimbal, which we call the sensor module. Additionally, two PAPRAS (Plug-and-Play Robotic Arm System) arms are mounted on top of the Spot robot, with Psyonic hands on its end-effectors. Finally, two computers are placed on Spot. The first is the Intel Jetson Xavier, and the second is a NUC. Details are shown in Fig. 1.

Fig 2: Communication diagram for football-catching pipeline

The full pipeline is shown in Fig. 2. In purple, a YOLO-based model is trained to robustly detect and localize a football. The model is run on the Jetson Xavier (which has a GPU for fast inference capabilities), which then uses its RGBD camera to calculate the football location in the 3D world. The football location is sent to the NUC, which controls Spot, the two robot arms and end effectors, as well as the 3-DOF sensor module.

My contributions for this project include the following:

  • Football detection model: Synthetic image generation, YOLO-based training, inference speed optimization techniques, implementation on the Jetson
  • Sensor module tracking algorithm (PD controller + post-processing algorithm)
  • Testing of the system in various environments
  • Exploring with high-speed camera integration

Football Detection Model

Synthetic Data Generation

(TODO: Explain why I did synthetic image generation) In order to create synthetic data, I first took pictures and videos of the football at various locations and orientations. Then, I segmented the football out of each image. This was done using the Segment Anything Model (SAM) [14]. The model version I used was the ViT-L SAM model. Using this model, I ran a Python script to create a football segmentation mask given a folder of images. The script is shown in Fig. 2.3, and the resulting segmentations are shown in Fig. 2.4. These segmentations were then used to construct synthetic data, where football segments were pasted into random images from Google’s Open Images Dataset V7. From there, synthetic images were generated, with bounding boxes automatically defined for training. An example can be seen in Fig. 2.5. The segmented footballs were added to the background with arbitrary rotation and also generated so as to have no occlusions. The scale was bounded above by how small the football should be from 5 meters away.

YOLO Model Training and Inference Techniques

The YOLOv8s model was used for detecting the football. Initially, the football detection model was the slowest component of the project. The model ran at around 10 frames per second on the Jetson, which is way too slow to get reliable estimates of the football location. To solve this issue, I used Nvidia TensorRT, which is an inference optimizer. Additionally, I converted the original model’s 32-bit floating-point numbers to 16-bit floating-point numbers. These two steps drastically increased the inference speed, reaching between 35-40 frames per second on the Jetson.
